## Security Policy — Visionary Termux

### Reporting Vulnerabilities in Integrated Scripts or Workflows

If you discover a security issue in:
- Scripts in this repository (`setup.sh`, `build_termux_package`, `termux-x11-preference`, `docs/snippets/`)
- GitHub Actions workflows (`.github/workflows/`)
- Imported integration content (`integrations/`)

Please open a **private security advisory** on this repository via GitHub's Security tab, or contact the maintainer directly.

### Upstream termux-x11 Security Issues

Security vulnerabilities in the core Termux:X11 Android application (rendering, IPC, NDK code) should be reported to the upstream project. See:

https://termux.dev/security

### General Termux Security

For security issues in Termux itself or its package ecosystem, see the upstream policy at the link above.
